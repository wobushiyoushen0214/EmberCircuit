# Ember Spire UI / VFX 参考审计

日期：2026-07-17

## 来源与边界

- 用户提供：`https://www.k399.games/app/ember-spire-555b`
- K399 详情接口指向独立 Web 游戏：`https://r3poorgmogmwy.beta-ok.kimi.link/`
- 本轮只读检查公开 HTML/CSS/JS，不登录、不提交、不下载或复制第三方代码/图像/字体。
- 只吸收视觉规律、交互节奏和可验证的性能/可访问性设计，不复制具体资产或版式。

## 可确认的视觉语言

- 主材质：暗炉、焦木、暗铁、黄铜、余烬；不是高饱和霓虹。
- 参考色：背景 `#0b0908/#12100e`、暖橙 `#e8622c`、黄铜 `#c9a45c`、危险红 `#a32633`、暖灰白 `#efe7da`。
- 面板：1px 半透明黄铜细边、四角短装饰线、内层高光/深色内阴影和柔和外投影。
- 卡牌：暗金属双层压边、焦角遮罩、按稀有度分级的局部辉光、hover 抬升。
- 地图：暗背景、虚线路径、菱形宝石节点、橙金可选态和暗红 Boss 外环。
- 确认存在的动效参数：受击高亮约 `300ms`、震动约 `260ms`、页面入场约 `320ms`、按钮按下缩放约 `0.96`、HP 延迟影子条约 `900ms`。

## 重要反证

发布 CSS 末尾明确关闭/隐藏灰烬粒子场、余烬轨道、能量呼吸、扫光、低血呼吸和角色脚底呼吸。因此不能声称参考游戏线上版本依赖大量持续粒子；其稳定优势是材质、层级、光影和短促状态反馈。

## EmberCircuit 映射

- 全局：深背景 + 底部余烬径向光 + 3%–5% 弱噪点 + 外围暗角。
- HUD：焦铁底、黄铜细边；HP 暗红、护甲灰蓝、能量暖橙、势能琥珀金；数值亮于标签。
- 卡牌：保留现有类型框，统一双层压边与焦角；hover `130ms`、上移 `24px`、缩放 `1.035–1.045`。
- 战斗：现有出牌飞行 `270ms` 保留；命中接 `60–90ms` hit-stop 和 `180–260ms` 受击恢复。
- Boss 阶段横幅：`180ms` 入场、`600–900ms` 停留、`200ms` 退场。
- 奖励：卡牌错峰 `40–60ms`；稀有奖励一次性 `500–700ms` 光效，禁止无限闪动。
- 页面转场 `220–320ms`；胜利余烬幕 `600–900ms` 且允许跳过。

## 性能与可访问性门

- 1280×720 常驻粒子总量建议不超过 40–60。
- 普通攻击 burst 12–18 粒子，Boss 技能不超过 30。
- 同屏模糊光斑不超过 6；避免多层全屏实时 shader。
- 新增“降低动态效果 / 闪光强度 / 粒子密度”。
- 降低动态模式关闭持续漂浮、呼吸和扫光，但保留短促状态反馈。
- 信息不得只靠颜色；正文对比度至少 4.5:1，战斗关键数字等效 16–18px。

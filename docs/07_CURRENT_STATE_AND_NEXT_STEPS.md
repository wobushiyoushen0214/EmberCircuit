# 当前完成度与下一阶段任务

## 当前完成度

项目现在是一个可运行的 Godot 4.x 卡牌肉鸽 MVP 骨架。

已具备：

- 数据驱动卡牌、敌人、遗物、遭遇、事件、状态、经济和地图配置。
- 可运行战斗核心。
- 敌人意图。
- 势能资源。
- 抽牌、出牌、弃牌、消耗。
- 护甲、生命、能量、状态。
- 战斗奖励三选一卡牌。
- 精英/Boss 遗物奖励。
- 战斗药水奖励。
- 问号事件选择。
- 第一章 10 个问号事件。
- 商店买卡、买药水和删卡。
- 篝火恢复和升级。
- 单槽位存档/读档。
- 分叉地图生成和地图节点选择。
- 分层地图视觉界面和路线连线。
- 牌组查看器和篝火升级预览。
- 占位音频事件。
- 药水槽、药水使用和药水存档。
- Boss 数据驱动转阶段和阶段意图切换。
- 战斗界面基础视觉样式、敌人占位美术、卡牌类型样式和药水图标。
- SVG 占位美术资源。
- Godot headless smoke tests。

## 仍未达到完整版的部分

- 没有正式美术、动画和受击反馈。
- 没有正式音效和 BGM。
- 地图视觉界面已有功能版，但缺少正式图标、美术主题和路线预览动效。
- 事件池仍缺少随机结果、前置条件和一次性排除规则。
- 没有多角色。
- 没有多章节。
- Boss 转阶段已有逻辑，但缺少阶段动画、音效和专属 UI 强提示。
- 没有 Steam/发布配置。
- 没有九宫格“移动壁垒”空间战术模式。

## 下一阶段建议顺序

1. Boss 表现强化：加入阶段动画、音效、震屏和阶段提示。
2. 战斗反馈：加入攻击闪烁、受击停顿、胜利/失败反馈。
3. 地图表现强化：替换节点图标、加入路线 hover 预览和节点详情面板。
4. 战斗美术替换：用正式卡图、敌人图和药水图标替换占位资源。
5. 事件系统升级：加入选择条件、一次性事件排除和随机结果。
6. 平衡模拟：批量自动跑战斗，输出平均回合数和死亡率。
7. 多章节路线：第二章节点、敌人池、Boss 和事件池。
8. 多角色：至少新增 1 个机制不同的角色。
9. 正式音频替换：用可授权音效和 BGM 替换临时音色。
10. 九宫格模式原型：新建独立 `GridCombatState`，不要污染当前战斗系统。

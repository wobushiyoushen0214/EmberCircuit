余烬回路 EmberCircuit
0.1.0-alpha.4 内部试玩版

一、运行方式

Windows：完整解压压缩包后运行 EmberCircuit.exe。当前内部版未做商业签名；若 Windows SmartScreen 拦截，请确认文件来自本次试玩包，再选择“更多信息 -> 仍要运行”。
macOS：解压后右键 EmberCircuit.app，选择“打开”。当前内部版未做 Apple 公证，首次启动可能需要在“系统设置 -> 隐私与安全性”中允许打开。

建议环境：Windows 10/11 x64 或 macOS 11 及以上，1280x720 或更高分辨率，键盘和鼠标。游戏会以最大化窗口启动。

二、主要操作

- 鼠标点击卡牌：使用卡牌。
- 鼠标拖拽卡牌到敌人或战斗舞台：选择目标并使用；右键或 Esc 取消拖拽。
- Tab：切换存活敌人。
- Space：结束回合。
- 数字键 1、2、3 或小键盘 1、2、3：使用对应药水槽。
- 设置页可调整音乐、音效、震屏、受击停顿和漂浮数字。

三、本轮建议重点体验

1. 分别使用余烬流亡者、电弧工匠和熔痕苦修者完成第一章，观察三个角色的卡组方向是否清晰。
2. 检查卡牌费用、伤害、护甲、势能和敌人意图是否容易理解。
3. 优先检查篝火：到达页应完整显示房间插画和“休息/锻造”，锻造页应一次展示牌组中全部可升级卡，页面不应出现可见滚动条。
4. 检查问号事件：插画、正文和全部选项在 1280x720 下应完整显示。
5. 尝试普通战、精英、事件、商店、篝火和宝箱路线，记录是否存在明显“必选”或“绝不选”的节点。
6. 尝试拖牌、点击出牌、快捷键和地图预览，记录任何卡顿、误触、文字裁切或控件重叠。
7. 有时间时完成三章，记录死亡章节、死亡遭遇、最终牌组和最强/最弱卡牌。
8. 试玩结束后，在“档案”、战败页或最终胜利页点击“导出报告”；成功后报告路径会复制到剪贴板。

四、反馈请包含

- 操作系统、分辨率和显示缩放比例。
- 使用角色、挑战等级、到达章节和大致游玩时长。
- 问题发生前的操作步骤。
- 截图或录屏。
- 对“最强卡牌、最弱卡牌、最难遭遇、最无聊节点”的判断。
- 附上导出的 ember_circuit_playtest_report.json；它包含本机最近 64 局匿名玩法记录。

五、当前已知限制

- 这是内部 Alpha，不是正式发行版。
- 位图与首版 SVG 美术仍混用；篝火房间和四个高频事件已使用正式候选位图，其余部分卡牌、敌人、事件和遗物画面尚未统一。
- 音乐和音效是第一版占位品质。
- 只有本地单槽位跑团存档，没有 Steam、云存档或跨设备同步。
- 平衡矩阵来自启发式自动玩家，本轮真人反馈会用于下一次数值修正。
- 真人试玩数据只保存在本机，不会自动上传；报告不含用户名、主目录、硬件序列号或网络标识，是否分享由试玩者决定。
- Windows 和 macOS 包均未进行商业代码签名，系统可能显示安全提示。

六、存档位置

Windows：%APPDATA%\Godot\app_userdata\EmberCircuit\
macOS：~/Library/Application Support/Godot/app_userdata/EmberCircuit/

发现无法继续游戏的严重问题时，请保留该目录中的 JSON 文件并连同复现步骤一起反馈。

真人试玩原始记录：ember_circuit_playtest_telemetry.json
可分享报告：ember_circuit_playtest_report.json
